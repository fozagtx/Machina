# Response Formats

## Output by Job Type

| Job Type | Files in Zip |
|----------|-------------|
| Build / Design | `index.html` + assets + `README.md` |
| Code / Algorithm | solution file + `README.md` |
| Copy / Tweet / Content | `response.md` (+ `design-notes.md` if brand work) |
| VSL Script | `vsl-script.md` with scene markers + timing |
| UGC Script | `ugc-script.md` with hook, body, CTA, B-roll notes |
| Research / Analysis | `response.md` with cited sources |
| Debug / Review | fixed code + `CHANGES.md` |

## Quality Standards

- ALL responses delivered as `.zip` — no exceptions
- Web projects: Tailwind CSS, Google Fonts, mobile-responsive, micro-animations
- Code: verified with `execute_code` before submission
- Copy / tweets: brand voice, ICP-aware, no filler text
- No TODOs, no stubs, no incomplete implementations

## Prohibited

- Financial advice presented as fact
- Harmful, illegal, or exploitative content
- Incomplete code with placeholder comments
